# Plan: API Reference Guide
**Goal:** Extract technical details into a structured API reference.

**Requirements:**
1.  **Output Location:** `docs/api/`
2.  **Files:** Create one file per major controller or module (e.g., `docs/api/auth.md`, `docs/api/users.md`).
3.  **Content:**
    - **Endpoint:** Method (GET/POST) and URL.
    - **Params:** Table of request parameters (Name, Type, Required, Description).
    - **Response:** JSON example of a success response.
    - **Errors:** List of common HTTP error codes.
